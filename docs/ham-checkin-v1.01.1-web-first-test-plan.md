# HAM 台网点名主控台 V1.01.1 Web-first 测试版试制说明

状态：试制规划  
目标版本：V1.01.1 测试版  
方向：网页主应用 + 可选轻量本地代理  
基线版本：V1.0.01

## 1. 背景判断

V1.0.01 已完成 Win64 安装包和 macOS 公证 DMG，但 Electron 桌面包体积偏大：Windows 安装包约 108 MB，macOS DMG 约 220 MB。当前业务代码本身并不大，体积主要来自 Electron 打包携带 Chromium、Node.js 和桌面运行时。

台网点名主控台的核心使用体验更接近网页应用：打开即用、快速登记、监听候选、导出 Excel / ADIF、呼号资料辅助补全。桌面版真正解决的问题主要是本地网络设备访问、离线保存和局域网权限，而不是界面本身必须运行在桌面壳中。

因此 V1.01.1 测试版建议转为 Web-first 试制：网页版作为主产品形态，桌面包降级为兼容方案，另行探索轻量本地代理以替代完整 Electron。

## 2. 版本目标

V1.01.1 不再以“继续增大桌面包能力”为主，而是验证以下方向是否可行：

- 网页版成为主入口：`https://fmo.bh1jss.net/checkin/`
- 本地设备访问从 Electron 壳中拆出，改为可选轻量代理。
- MMDVM / HAMBOX 的局域网读取能力通过代理试制；FMO 本地代理先预留锚点，不在公网测试阶段默认开放。
- 用户数据仍优先保留在浏览器本地，可导出 Excel / ADIF。
- 桌面安装包仅作为 fallback，不再作为主推下载。

## 3. 产品形态

### 3.1 网页主应用

网页主应用继续使用现有 Vue 页面，保留 V1.0.01 已完成能力：

- 台网活动信息录入。
- 备选区 4 卡片工作流。
- 右侧最近 6 个呼号高亮。
- MMDVM TS1 / TS2 时隙筛选。
- HAMBOX、BM DMR、FMO 候选源。
- Excel / ADIF 导出。
- 呼号资料库辅助补全。
- 中英文说明书。
- 匿名使用统计提示。

### 3.2 轻量本地代理

本地代理只做一件事：让经过授权的网页安全读取用户局域网内的设备数据。公网网页入口默认不开放本地设备访问；如需开放，必须先完成注册并由作者审核通过。

建议默认监听：

```text
http://127.0.0.1:37174
```

建议提供接口：

```text
GET /health
GET /proxy/mmdvm?host=192.168.31.119
GET /proxy/hambox?host=192.168.31.120
```

代理职责：

- 只允许访问局域网或用户显式填写的设备地址。
- 公网网页调用本地代理前必须检查授权状态。
- 增加超时和错误提示。
- 返回标准 JSON 或原始 HTML 解析结果。
- 设置允许 `https://fmo.bh1jss.net` 调用的 CORS。
- 不保存用户日志，不上传设备数据。

### 3.3 桌面版定位

V1.0.01 Electron 桌面版继续保留，但在 V1.01.1 中定位为：

- 离线环境 fallback。
- 不方便配置本地代理的用户 fallback。
- 长期活动现场的稳定备选方案。

首页和说明书应逐步把“网页版 + 本地代理”作为推荐路径，把 Electron 桌面包放到“备用下载”。

## 4. V1.01.1 最小试制范围

第一阶段不要做大而全，只验证最关键链路：

1. 网页端增加“本地代理模式”选项。
2. 网页端可检测 `127.0.0.1:37174/health`。
3. MMDVM 本地读取优先尝试本地代理。
4. HAMBOX 本地读取优先尝试本地代理。
5. 公网网页未通过注册审核时，不允许启用本地代理。
6. 代理未运行时给出清晰提示，而不是直接报连接失败。
7. 保留当前 Electron 桌面包作为回退，不删除旧路径。

暂不纳入第一阶段：

- 自动发现局域网设备。
- 公网网页直接开放 FMO 本地代理。
- 代理自动升级。
- 后台常驻托盘。
- Windows 服务安装。
- macOS LaunchAgent 安装。
- 完整设备配置向导。

## 5. 轻量代理技术候选

### 5.1 Node.js 单文件代理

优点：

- 与现有 `server/index.mjs` 代码可复用。
- 试制最快。
- 可直接复用 MMDVM / HAMBOX proxy 逻辑。

缺点：

- 如果用户没有 Node.js，需要额外打包。

### 5.2 Go 或 Rust 小代理

优点：

- 可编译成小体积单文件。
- 不需要用户安装运行时。
- Windows / macOS 体积可显著小于 Electron。

缺点：

- 需要重写代理层。
- 打包签名流程另起一套。

### 5.3 Tauri 小壳

优点：

- 体积比 Electron 小。
- 可以保留桌面感。

缺点：

- 仍然是桌面壳，不是最纯粹的 Web-first。
- 对当前“只需要代理”的目标可能仍偏重。

建议 V1.01.1 先用 Node.js 代理试制，验证产品路径；验证成功后再评估 Go/Rust 单文件代理。

## 6. 前端改动锚点

建议新增配置项：

```js
localProxyEnabled: true
localProxyUrl: 'http://127.0.0.1:37174'
```

建议 UI 位置：

- 右侧监听源设置区增加“本地代理”状态。
- 当选择 MMDVM / HAMBOX 本地源时显示代理检测状态。
- 公网网页未获得作者审核通过时显示“网页版本地设备访问需注册并通过作者审核”。
- 代理在线：显示“本地代理已连接”。
- 代理离线：显示“未检测到本地代理，可使用桌面版或启动轻量代理”。

建议数据流：

```text
网页版
  -> 检测 127.0.0.1:37174/health
  -> 公网入口先检查注册审核状态
  -> 代理在线：通过代理读取 MMDVM/HAMBOX
  -> 代理离线：使用现有网页能力或提示 fallback
```

## 7. 测试矩阵

| 场景 | 验证内容 | 阻断标准 |
| --- | --- | --- |
| 纯网页 BM DMR | 不安装代理也能打开网页并读取 BM | 网页打不开或 BM 基础流程失败 |
| 公网网页未授权 | 选择 MMDVM/HAMBOX 时提示需注册审核，不能启用代理 | 未授权也能读取局域网设备 |
| 网页 + MMDVM 代理 | `192.168.31.119` Last Heard 可读取，TS1/TS2 筛选有效 | 无法读取或筛选错误 |
| 网页 + HAMBOX 代理 | HAMBOX 最近通联可读取 | 无法读取候选 |
| 代理未启动 | 网页给出清晰提示，不崩溃 | 用户只看到底层网络错误 |
| Excel / ADIF 导出 | 网页端仍可导出 | 文件损坏或字段缺失 |
| macOS 浏览器 | Safari / Chrome 访问网页与代理 | 本地代理被浏览器阻断且无说明 |
| Windows 浏览器 | Edge / Chrome 访问网页与代理 | 无法连接代理 |

## 9. 回退策略

V1.01.1 是试制版，不能影响 V1.0.01 已可用路径：

- `https://fmo.bh1jss.net/checkin/` 保持可用。
- V1.0.01 Win64 安装包和 macOS DMG 保留下载。
- 本地代理功能可通过前端开关关闭。
- 如果代理试制不稳定，首页仍可回退到 V1.0.01 桌面包推荐模式。
