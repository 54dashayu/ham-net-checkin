# HAM 台网点名主控台 V0.9.01 使用说明书

版本：V0.9.01  
适用平台：网络版、Win64版、MacOS版  
网络版入口：`https://fmo.bh1jss.net/checkin/`  
下载页：`https://fmo.bh1jss.net/`

## 1. 软件用途

HAM 台网点名主控台用于业余无线电台网点名活动。主控可以在活动中记录友台呼号、时间、QTH、设备、天线、功率、模式、信号报告和备注，并导出 Excel 台网日志。

软件的核心原则是“监听只做候选，记录必须由主控确认”。右侧监听区提供最近通联候选，左侧记录区才是正式点名日志。

## 4. 主界面概览

主界面分为四个区域：

1. 顶部活动信息区。
2. 左侧主控记录区。
3. 右侧实时监听候选区。
4. 下方已记录区和主控发射提示。

建议工作流程：

1. 填写顶部活动和主控信息。
2. 选择监听源并连接。
3. 从右侧候选中点选友台，或手工输入呼号。
4. 检查 QTH、设备、天线、功率、模式和信号报告。
5. 点击加入记录。
6. 活动结束后保存或导出 Excel。

## 5. 顶部活动信息区

顶部字段包括：

- 台网活动名称。
- 主控呼号。
- 主控 QTH。
- 主控设备。
- 主控天线。
- 主控功率。

这些信息会写入 Excel 顶部主控信息行。

顶部还显示：

- 已记录数量。
- 下一条记录序号。
- 秒级系统时间。

常用按钮：

- 保存：保存当前点名记录。
- 自动保存：开启后软件会自动保存。
- 新建：新建一个空白点名活动。

### 已记录数量和序号

如果活动开始前已经有纸质或其它系统记录，可以设置已记录数量。软件下一条记录会从该数量继续递增。

例如已记录数量为 `42`，下一条序号会从 `43` 开始。

## 6. 主控记录区

主控记录区用于填写最终入表记录。

字段包括：

- 前缀。
- 呼号。
- 时间。
- QTH。
- 设备。
- 天线。
- 功率。
- 模式。
- 信号报告。
- 备注。

### 呼号输入

软件会对呼号进行标准化：

- 主呼号输入框用于输入核心呼号。
- 前缀可单独输入。
- 输入数字前缀时，可自动转换为类似 `B3/` 的前缀。
- 支持带前缀和后缀的核心呼号匹配。

建议：

- 主呼号框只输入核心呼号，例如 `BH1JSS`。
- 移动、区域或跨区前缀后缀尽量使用前缀/备注方式记录。

### 呼号候选

输入呼号片段时，软件会从历史记录和呼号资料库中显示候选。点击候选可快速填入呼号和相关资料。

### 资料补全

QTH、设备、天线、功率、模式、信号报告支持补全：

- 默认优先显示当前呼号历史用过的资料。
- 如果在字段中输入片段，可从全库模糊匹配。
- 新呼号会提示“首次参与”。
- 曾经参与过的呼号会显示历史参与次数。

### 添加记录

确认资料无误后，点击加入点名表。记录会进入已记录区，序号自动递增。

### 编辑记录

在已记录区点击编辑，可把记录带回左侧表单修改。修改后保存即可更新原记录。

### 删除记录

支持单条删除、选择多条删除和清空本次记录。清空前建议先导出 Excel 或 JSON 备份。

## 7. 实时监听候选区

实时监听候选区用于显示最近通联台站。候选不会自动进入点名记录，必须由主控手工点选或确认。

候选卡片通常显示：

- 呼号。
- 时间。
- QTH。
- 设备。
- 功率。
- 模式。
- 来源或中继信息。

点击候选后，软件会把可用信息填入左侧记录表单。

## 8. 监听源说明

### FMO

适合本地版使用。填写 FMO 地址后，软件会连接实时事件和控制接口。

地址示例：

```text
192.168.40.3
192.168.40.3:40088
fmo.example.net:40088
```

协议：

- 局域网设备通常使用 `ws`。
- 公网 HTTPS 环境可尝试 `wss`。

注意：

- 网络版受浏览器安全策略限制，通常不适合访问本地 FMO。
- 如果控制接口不可用，软件会尝试使用实时监听作为候选来源。

### MMDVM

适合本地版使用。软件读取：

```text
http://设备地址/mmdvmhost/lh.php
```

用于显示 Last Heard 候选。

注意：

- Last Heard 不等同于“正在发射”。
- MMDVM 设备必须能被电脑直接访问。

### HAMBOX

适合本地版使用。软件读取本地 HAMBOX Dashboard/Last Heard 数据。

注意：

- 设备地址必须能从电脑访问。
- 若 HAMBOX 返回 transmitting 状态，软件会优先使用。

### BM DMR

适合网络版和本地版使用。填写 BrandMeister 通话组，例如：

```text
46001
```

软件会监听指定 TG 的实时数据，并尝试补全 BM 设备资料。

### 其它来源

YSF、FCS、D-Star / XLX、P25、NXDN 当前保留入口。V0.9.01 主要完成入口和后续适配基础，实时监听接口未完整开放。

## 9. 主控发射提示

底部主控发射提示用于提醒当前实时列表首位是否为主控呼号。

规则：

- 取实时监听列表首位候选。
- 与顶部“主控呼号”做核心呼号匹配。
- 首位匹配主控时提示“正在发射”。
- 首位不再是主控时恢复等待状态。
- 对没有明确结束事件的来源，会使用短时间失活兜底。

该提示仅作为现场辅助，不参与正式记录。

## 10. 已记录区

已记录区展示本次活动正式点名记录。

功能：

- 倒序显示，新记录在上方。
- 搜索呼号、QTH、设备和备注。
- 编辑记录。
- 删除记录。
- 多选。
- 全选。
- 批量删除。

建议活动中定期保存，活动结束后导出 Excel 和 JSON 备份。

## 11. 保存与导出

### 保存

点击顶部“保存”会保存当前点名表格。

本地版：

- 可保存到本地 Excel 文件。
- 支持浏览器文件系统能力时，可持续写入同一文件。

网络版：

- 保存到服务器。
- 服务器生成 Excel 文件。
- 受网络版测试限制约束。

### 自动保存

开启自动保存后，记录变化会触发自动保存。活动现场建议开启。

### Excel 导出

Excel 文件为 `.xlsx` 格式，兼容 Excel 和 WPS。

Excel 包含：

- 活动名称。
- 主控呼号、QTH、设备、天线、功率。
- 点名记录。
- 页脚说明。

### JSON 备份

JSON 备份用于保留软件内部数据，适合：

- 活动前后备份。
- 换电脑。
- 回滚恢复。
- 给开发者排查问题。

JSON 备份不是正式台网日志，正式归档建议使用 Excel。

## 12. 呼号数据库

呼号数据库用于辅助补全 QTH、设备、天线、功率、模式、信号报告等资料。

### 本地历史库

每次保存记录后，软件会从当前记录中积累呼号资料。下次输入同一呼号时，会优先显示该呼号历史资料。

### 共享呼号数据库

共享库需要启用和注册。

入口：

- 启用呼号数据库。
- 注册。
- 同步。

注册资料包括：

- 注册呼号。
- CRAC 操作证书号。
- 常用 QTH。
- 常用服务器。

审核通过后，作者发放验证密钥文件。导入验证密钥后，即可同步共享呼号资料库。

### 验证密钥

验证密钥用于启用共享库。请妥善保存，换机或重装后可重新导入。

可用操作：

- 导入验证密钥。
- 导出验证密钥。

## 13. 旧数据库导入

软件支持导入原 Windows 点名软件 `.db3` 文件。

用途：

- 迁移旧点名记录。
- 生成历史呼号画像。
- 复用旧库中的 QTH、设备、功率和备注资料。

导入后建议检查记录数量和关键呼号资料，再进行正式活动。

## 14. 新建活动

点击“新建”可创建空白活动。

注意：

- 当前点名记录会保留在本地历史中。
- 新活动从空白记录开始。
- 建议新建前先保存或导出当前活动。

## 16. 常见问题

### 为什么网络版不能稳定访问我的 FMO 或 MMDVM？

浏览器中的网页受 HTTPS、mixed content、CORS 和局域网访问策略限制。需要稳定访问本地设备时，请使用 Win64版或 MacOS版。

### 为什么监听列表里的候选不会自动入表？

这是刻意设计。监听数据可能有误、可能是上一轮通联、也可能不是主控抄收清楚的友台。正式点名记录必须由主控确认。

### 为什么 MMDVM Last Heard 里有呼号但不一定显示正在发射？

Last Heard 是最近听到，不等于当前仍在发射。软件不会把 Last Heard 永久当成正在发射状态。

### macOS 还会提示安全吗？

V0.9.01 MacOS版已完成 Developer ID 签名、Apple 公证和 staple。正常下载应显示为受认可开发者来源。如果仍有提示，请确认下载的是 `HAM-Checkin-0.9.01-macOS-Universal-notarized.dmg`。

### Win64 为什么是 ZIP？

公开网页直接分发 EXE 可能被服务器或浏览器安全策略拦截。V0.9.01 使用 ZIP 分发，解压后运行里面的 EXE。

### 如何确认下载文件没有损坏？

下载校验文件 `SHA256SUMS-HAM-Checkin-0.9.01.txt`，对照本地文件 SHA256。

3. 设置已记录数量。

2. 导出 Excel。
